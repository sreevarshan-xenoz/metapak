Robustness & Safety Design

Architecture for the "Never-Break" System Suite

Component Overview

1. Snapshot Engine

Generic Trait Interface:

  • create_snapshot(name)
  • rollback(id)
  • list_snapshots()
Providers: BTRFS, ZFS, LVM, Timeshift

2. Simulation Layer

Pre-Execution Hook:

  • Native Dry-Run (Pacman -S p)
  • Disk Space Projection
  • Conflicting File Detection

3. Health Watchdog

Background Checks:

  • Mirror Latency Monitoring
  • GPG Key Expiry Alerts
  • Database Integrity Verifier

The Workflow

  1. User selects packages and hits Install.
  2. Simulation Layer runs a dry-run. If conflicts found → Show Warning.
  3. Snapshot Engine creates a "Pre-Install" restore point.
  4. Execution proceeds. If failure occurs → Offer 1-click Rollback.
