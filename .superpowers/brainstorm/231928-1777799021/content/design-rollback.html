The Rollback Strategy

  • Automatic cleanup: If an operation succeeds, the "Pre-Install" snapshot is marked for auto-deletion after 24 hours (configurable).
  • Forced Rollback: If the user chooses Rollback, the system reboots (if needed for root FS) and uses the snapshot provider to set the default subvolume/dataset to the pre-install state.
  • Logging: Every rollback event is logged with full context (why it failed, what was being installed).
