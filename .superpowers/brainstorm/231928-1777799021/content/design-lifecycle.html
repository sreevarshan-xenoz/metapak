The Transaction Lifecycle

Ensuring 100% safety from start to finish

Pipeline Flow
Step 1: Health Check
Ping mirrors, verify GPG signatures, check database locks.
Step 2: Simulation
Run dry-run. Calculate disk usage. Parse conflict warnings.
Step 3: Atomic Snapshot
Create system restore point (e.g., arch-tui-pre-install-firefox).
Step 4: Execution
Run the actual command (pacman, apt, etc.).
Step 5: Failure Handling
If exit != 0, show the Rollback Dialog.

Safety Checkpoints

The system will automatically cancel if:

  • Mirror latency exceeds 2000ms.
  • GPG keys are invalid/expired.
  • Available disk space < Package size + 500MB buffer.
