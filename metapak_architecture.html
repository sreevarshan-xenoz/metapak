📋 Architecture Pattern

Provider + Event Loop + Orchestrator — metapak uses a layered architecture with a central event loop driving TUI rendering, built-in providers for 5 package managers (Pacman, AUR, npm, Cargo, pip) behind a unified PackageProvider trait, and an orchestration layer coordinating parallel operations, transactions, and safety guarantees.

🎯 Design Principles

🔌 Provider Abstraction

All 5 providers implement the unified PackageProvider trait. Providers are built directly into services.rs — no separate backend files needed.

🛡️ Fail-safe Transactions

Snapshot → Execute → Verify → Rollback-on-failure → Post-snapshot pipeline ensures system integrity during package operations.

⚡ Parallel Async Execution

All 5 providers queried concurrently via tokio tasks. Search, install, and remove operations run in parallel.

⏱️ Debounced Search

300ms configurable debounce window prevents flooding providers with intermediate keystrokes during interactive search.

🔁 Circuit Breaker Pattern

AUR API circuit breaker monitors error rates, auto-disables queries on threshold exceeded, and periodically attempts recovery.

🧩 Separation of Concerns

Each source file has a focused responsibility. Clear boundaries between UI, state, orchestration, providers, and infrastructure layers.

🛡️ Safety Patterns

⏰ Command Timeout

All subprocess executions have configurable timeout handling

🔁 Auto Retry

Automatic retry logic for DB locks and dependency errors

🛡️ Permission Validation

Security checks before subprocess calls

🔍 Operation Simulation

Dry-run preview before committing changes

📸 Snapshot Rollback

Filesystem snapshots with automatic rollback on failure

🧹 Input Sanitization

Sanitized user input for safe subprocess execution

📊 Structured Errors

Typed error hierarchy for precise error handling

🔌 API Circuit Breaker

External API protection against outages

📋 Transaction History

Persistent structured logs of all operations with rotation

🏗️ Detailed Architecture Layers

🏁 1. Entry Layer

src/main.rs — Clap-based CLI argument parsing, config initialization, terminal setup (raw mode, alternate screen), and the main event loop via App::run().

🖥️ 2. UI + Input Layer

All TUI rendering and keyboard interaction. The App receives keyboard events, transforms them to Actions via keybindings, and dispatches them.

ui.rs ~1500 lines of Ratatui rendering
input.rs Key event parsing
keybindings.rs Key → Action mapping
action.rs Action enum (all user actions)
ui_utils.rs Rendering helpers, progress bars
animations.rs Loading spinners, transitions
🧠 3. Application State Layer
app.rs
  • App — Central orchestrator holding all state
  • AppMode — Current mode (Search, Browse, Detail, Help, etc.)
  • App::run() — Main event loop
  • App::handle_action() — Action dispatcher
  • App::fetch_all_packages() — async package loading
🔀 4. Orchestration Layer
services.rs
  • PackageService — Unified search/install/remove
  • 5 built-in providers
  • CircuitBreaker for AUR API
parallel.rs
  • run_parallel() — Concurrent tokio tasks
  • Cancellation support
operation_queue.rs
  • OperationQueue — Batch operation orchestration
  • OperationPreview — Summary generation
⚙️ 5. Business Logic Layer
search.rs Fuzzy matching, query parsing, scoring
  • EnhancedSearch, SearchQuery, FuzzySearch
  • AND/OR/NOT operators + filters
simulation.rs Dry-run simulation
  • PackageSimulator, simulate_upgrade()
transaction_manager.rs Safe transactional pipeline
  • TransactionManager, TransactionConfig
dependency_visualization.rs Interactive dependency tree
  • DependencyVisualizer, DependencyNode
🔌 6. Provider Adapters Layer

5 package providers built into services.rs + 2 snapshot providers, all behind trait interfaces. traits.rs defines PackageProvider (search, is_installed) and UpdateProvider (check_updates, get_outdated_packages).

⚙️ Configuration

Path: ~/.config/metapak/config.toml
Format: TOML
Migration: Auto-migration from legacy ~/.config/arch-tui/config.toml

[general]
  • debounce_delay_ms — u64, default 300
  • max_results — usize, default 100
  • refresh_rate — u64, default 60
  • cache_ttl_seconds — u64, default 60
[search]
  • cache_ttl_seconds — u64, default 60
[snapshot]
  • enabled — bool
  • provider — btrfs | timeshift
  • create_before / create_after — bool
  • max_snapshots — u32
  • snapshot_prefix — String
[hooks]
  • pre_install, post_install — Vec<String>
  • pre_remove, post_remove — Vec<String>
  • pre_update, post_update — Vec<String>
[telemetry]
  • enabled — bool
  • log_dir — String
  • max_log_files — u32
  • max_log_size_mb — u32
[color]
  • theme — 'mocha' | 'latte'
